Hexo 學習心得

Documentation

環境需求

  • 需安裝:
*    **Node.js**
*    **git**
  • 檢查Node.js

QuickStart

  • step1: 透過npm去下載Hexo，只需下令一次，就會安裝hexo在系統裡面並且幫你配置global環境變數
  • step2: 在自己安排的位置下初始化接下來要操作的資料夾
  • step2’: 也可以在當前目錄下初始化，這樣會使當前目錄變成專案資料夾
  • step3: 在被init過的目錄裡面執行安裝
  • step4: 啟動blog
  • step4’: 如果port number有問題
  • step5: 設定Github pages
* repo名稱設為**username\.github\.io**
* branch設為main, 跟目錄/(root)
  • step6: 在_config.yml編輯deploy:
  • step7: install git相關檔案
  • step8: 產生static file
  • step9: 部署到你的repo
  • step10: 進入 username.github.io 查看blog

檔案架構

  • _config.yml
*    更改設定的地方
  • package.json
*    Application data
  • scaffolds
*    當生成新文章後，hexo會參考此資料夾的形式去生成
  • source
*    contents放這裡
  • themes
*    hexo會把themes和contents做合併，產生靜態網站的檔案**public**
