初入 Hexo

前言

  • 非專業的文件
  • 有簡易的執行步驟與人性化的說明
  • 專注在開發與踩雷心得
  • 沒有 IDE, Node.js, Git的安裝教學(其實安裝都很簡單啦)
  • 示例環境為 Windows 11, VSCode

為甚麼選擇 Hexo

  • HTML/CSS/JavaScript 前端三寶我只會基礎中的基礎而已，更遑論使用框架開發
  • 我很想要酷酷的個人網站
  • 練習看 tool 的文件並執行開發，因為我之前會對於看 documentation 感到恐慌，不可以再逃避了
  • 有些系上的大佬也是用 Hexo 架設 blog，想走看看強者走過的路
  • 有這麼方便的 tool，而且成果也會比自己想像的壯麗，還有不玩看看的理由嗎?

Hexo 簡介

  • A fast, simple & powerful blog framework, powered by Node.js.
    • 抄官網的啦XD
  • 只需要透過下指令、更改 configuration 就可以開發出個人網站，就是不需要打 code 啦!
  • 產生的網頁為靜態網頁，沒有後端的系統，所以網頁加載速度很快
  • 撰寫文章是透過 markdown 語法去實現
    • 語法簡單
    • 可以寫出流程圖、數學函式等純文字文件難以實現的內容

Hexo 開發

(一) 準備開發 Hexo 環境

安裝所需的軟體

  • (optional)安裝 Visual Studio Code ，用以編輯設定檔以及撰寫文章
  • (required)安裝 Git ，用以部署到 Github，透過 Github 幫你代管
  • (required)安裝 Node.js ，因為需使用到 npm 去安裝部分檔案

建置環境

  • 使用 cmd 在任意目錄下執行，把 hexo-cli 下載在 global 端，也就是系統中的所有使用者都可以使用 hexo
  • 這樣子環境就準備好了，接下來就是開發你的專案!!!

(二) Quick Start

  1. 準備一個放專案的目錄，目前就先預設此目錄名字為 blog-projects
  2. 打開你的cmd並且進入 blog-projects 目錄下
  3. 初始化你的第一個專案，MyBlog 可以換成你喜歡的名字
  4. 這時候 MyBlog 就是你創建出來的一個專案了!!!
  5. 進入 MyBlog 專案內
  6. 安裝必要的檔案
  7. 安裝完成後就是: 網頁~啟動~

(三) 專案結構基本說明

只提及接下來會運用到的檔案或目錄

  1. _config.yml : 用來配置你的 blog，看看註解就能略知一二
  2. source : 網站的原始內容放在這邊，圖片、PDF 檔、文章等。
  3. themes : 放置你所下載的主題
  4. public : 產生的靜態檔案會放在這裡

(四) 設定Blog主題

如何下載心儀的主題?

  1. 首先在 Themes | Hexo 選擇你喜歡的樣式
  2. 接著點進該主題的 Github，基本上都有配置文件，照著文件說明去安裝即可
    • 其實就是 git clone 而已
  3. 下載完成後你就會看到 themes 就有該主題的資料夾了

調整主題所提供的項目

接下來以 YourTheme 代稱你所選擇的主題，以便簡化說明

  1. 首先調整 _config.yml 之中的theme，將冒號右側改成你要的主題
  2. 打開 yourtheme 的資料夾
  3. 複製其中的 _config.yml 的所有代碼
  4. 在MyBlog之下新增 _config.yourtheme.yml 並且把你複製的代碼貼到此檔案之中
  5. 之後就是看官方文件以及不斷微調 _config.yourtheme.yml 其中的內容
  6. 由於不同主題的 config 會不同，效果也不同，接下來就不細說，建議直接看主題的官方文件，應該都很詳細。你會越來越了解 config 怎麼調整ㄉ
  7. 編輯 _config.yourtheme.yml 的過程中，請時時使用 hexo server 查看效果如何，記得不要一次改太多喔!

(五) Hexo基本操作

  • 初始化一個專案， <your_project_name> 換成你喜歡的名字
  • 產生靜態網頁檔案，放在 public 裡面
  • 在 local 端啟動網頁，讓你可以查看網頁目前的模樣
  • 刪除 public 以便後續更新網頁並重新生成、部署網頁
  • 將靜態檔案部署到你指定的 repository 中
  • 產生一則新的貼文， your_post_name 換成你要發的文章名
  • 產生一個新的頁面， your_page_name 換成你指定的頁面名稱

(六) 部署到Github

  1. 註冊一個 Github 帳號
  2. 創建一個新的 repository 為 public，並且命名為 username.github.io ， username 換成你的用戶名稱

(七) 你可能會有的問題

  1. 你可能會看到這個 error: ERROR Plugin load failed: hexo-site
  • 只要把 node_modules 之下的 hexo-site 刪除即可，就不會有錯誤訊息了
  1. 為甚麼不直接調整 yourtheme 之下的 _config.yml 呢?
  • 為了避免更新主題後，客製的項目被覆蓋掉回到預設值
  • 至於命名一定要是那樣，這樣才能被 hexo 專案找到
  1. 在調整主題的 config 時，會看到 fa-solid fa-face-smile ，這啥呢?
  • 這是在引用 Font Awesome 圖示庫裡面的 CSS 樣式
  • fa 就是 Font Awesome， solid 是風格， face-smile 是笑臉
